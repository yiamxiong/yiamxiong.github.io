Systems Analysis & Design Process Overview

1. ERD Creation (Entity Relationship Diagram)

Modeled the initial relationships between Customers, Orders, Employees, and Parts using Draw.io. Focused on key attributes and cardinality to reflect real-world interactions.

2. Normalization to 3NF

Refined the diagram into a 3rd Normal Form structure, eliminating redundant data and ensuring every non-key attribute is fully dependent on the primary key. This promotes scalability and data integrity.

3. Relational Schema Design

Translated the normalized model into relational tables using Microsoft Access. Primary and foreign keys were applied to enforce referential integrity.

4. Sample Data Entry

Input example records into each table to validate structure and test query results. This step helped verify that the design supported real-world scenarios and business logic.
